RedisTemplate设置setEnableTransactionSupport(true)后，在@Transactional注解的service里查询，返回结果为null

发表于 2019-11-06 | 分类于 问题
字数统计: 495 | 阅读时长 ≈ 1 分钟

  项目测试发现一个奇怪的现象：
在service用RedisTemplate查询某个key的缓存，事先通过redis gui可以确定redis中确实存在该key，但是RedisTemplate始终返回的是null。

调试过程中，我在controller中注入RedisTemplate，并使用service里同样的代码查询，却可以正常返回值。

也就是说：使用同一个RedisTemplate，查询同一个key，在controller里可以查到值，在service里却返回null？

  查看controller和service代码并没有发现有什么不一样的地方，除了service类头上加了一个@Transactional注解搞的事呢？
注解。是不是这个@Transactional注解搞的事呢？

首先来看看redis的配置文件：

没什么大问题，看起来有可能出问题的只有template.setEnableTransactionSupport(true);这行代码。 平时项目中我一般并不会配置这一句，上次看到就鬼使神差的加上了这句话，并没有过多的追究这行代码的作用。看字面意思应该是开启redis的事务支持。

通过查询得知：
redis事务将命令放到Queue中，因为命令并没有执行，所以无法拿到返回值。

比如：

redisTemplate是无法拿到返回值的，因为redis在MULTI/EXEC代码块中，命令都会被delay，放入Queue中，而不会直接返回对应的值。所以例子中的increment自增,一般自增命令执行成功后默认会返回自增之后的值，但这里在redis事务中却是返回了null。

既然找到原因，那么把template.setEnableTransactionSupport(true);这行代码去掉应该可行，一试果然拿到返回值了。
