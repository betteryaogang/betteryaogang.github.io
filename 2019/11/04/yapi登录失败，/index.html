mongodb进程意外终止，再次启动报错：Job for mongod.service failed because the control process exited with error code. See "systemctl status mongod.service" and "journalctl -xe" for details.

发表于 2019-11-04 | 分类于 问题
字数统计: 505 | 阅读时长 ≈ 2 分钟

  前段时间搭建了YApi平台来作为项目的接口管理平台，几个月下来运行都还挺稳定的。今天小伙伴突然跑来跟我说Yapi平台登录不上了。

我试了一下发现果然登录不上了，F12打开控制台，login接口返回 登录失败...,没有其他多余信息。

于是登录服务器，尝试重启Yapi服务，故障依旧。

能够打开登录页面，说明Yapi服务本身没问题，登录不上很有能是Yapi平台到数据库不通。一查进程果然，mongodb进程不知道什么时候被kill了。

执行systemctl start mongod.service 启动mongodb

发现并不能启动起来，报错：
Job for mongod.service failed because the control process exited with error code. See "systemctl status mongod.service" and "journalctl -xe" for details.

于是决定看看mongodb日志 /var/log/mongodb/mongod.log,发现日后最后有这么一段：

No space left on device,问题就很明显是服务器盘满了，服务器是40G的默认盘，程序跑久了日志就把空间占的差不多了，清理掉无用的日志文件，重启mongodb、重启Yapi，问题解决。
